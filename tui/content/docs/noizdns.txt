# NoizDNS

DPI-resistant fork of DNSTT. Adds noise and padding to
DNS queries to evade deep packet inspection.

## Why NoizDNS

- Resists DPI fingerprinting of DNS tunnel patterns
- Adds random padding to defeat traffic analysis
- Same unblockable DNS transport as DNSTT
- Better stealth than vanilla DNSTT

## When to Use

- DNSTT is being detected by DPI
- Censors actively fingerprint DNS tunnels
- Need maximum stealth over DNS
- Total internet blackout scenario

## How It Works

Builds on DNSTT but adds random noise and padding to DNS
queries, making traffic analysis much harder.

  Client --> DNS Resolver --> Your NS Server
    |            |               |
    |  DNS (53)  |  Noisy/padded |
    |  +padding  |  DNS queries  |

## Setup Notes

- Requires domain with NS record pointing to your server
- Uses port 53 (DNS) -- conflicts with DNSTT and Slipstream
- Only one DNS tunnel protocol can be active at a time
- Throughput similar to DNSTT (~42 KB/s)
- Better DPI resistance at cost of slightly more overhead
